🧪 Unit Testing en .NET

Fundamentos y mejores prácticas

Johan Bueno | 2026

¿Qué es Unit Testing?

  • Prueba unitaria = Prueba de la unidad más pequeña de código
  • Aisla componentes individuales (métodos/funciones)
  • Verifica comportamiento esperado
  • Automatizado y repetible
  • Ejecutable en cualquier momento
Objetivo: Detectar errores temprano, mejorar calidad del código

¿Por qué Unit Testing?

  • ✅ Detecta bugs antes de producción
  • ✅ Facilita refactoring con confianza
  • ✅ Documenta el comportamiento esperado
  • ✅ Mejora la calidad del código
  • ✅ Reduce costos de mantenimiento
  • ✅ Aumenta velocidad de desarrollo a largo plazo

Patrón AAA

Arrange - Act - Assert

Arrange: Preparar datos y objetos necesarios

Act: Ejecutar la acción/método a probar

Assert: Verificar que el resultado es el esperado

// Arrange
int a = 5, b = 3;
// Act
int resultado = Calculadora.Sumar(a, b);
// Assert
Assert.Equal(8, resultado);

Frameworks Populares

  • xUnit.net - Moderno, flexible (recomendado)
  • NUnit - Clásico, muy usado
  • MSTest - Framework de Microsoft
Librerías complementarias:
  • Moq - Mocking de dependencias
  • FluentAssertions - Assertions más legibles
  • Bogus - Generación de datos de prueba

Test Básico con xUnit

using Xunit;
public class CalculadoraTests {
[Fact]
public void Restar_NumerosPositivos_RetornaResultadoCorrecto() {
var calc = new Calculadora();
var resultado = calc.Restar(10, 3);
Assert.Equal(7, resultado);
}

Attributes en xUnit

  • [Fact] - Test sin parámetros
  • [Theory] - Test parametrizado
  • [InlineData] - Datos para Theory
  • [Skip] - Saltar test
  • [Trait] - Categorizar tests

Mocking: Aislar Dependencias

Mock = objeto simulado que reemplaza una dependencia real

FluentAssertions

Mejora la legibilidad de los assertions

Cobertura de Tests

  • % de código ejecutado por tests
  • Meta recomendada: 70-80%
  • NO es igual a calidad

Mejores Prácticas

  • ✅ Una aserción principal por test
  • ✅ Nombres descriptivos
  • ✅ Mantener tests independientes

🎯 Resumen
